# Политика безопасности

## Общие правила

При работе с репозиторием **строго запрещается**:

1. **Коммитить секреты**: пароли, токены, API-ключи, приватные SSH-ключи.
2. **Загружать персональные данные третьих лиц** без их согласия или без должной анонимизации.
3. **Использовать закрытые или коммерческие датасеты**, на которые у вас нет прав.

## Рекомендации

- Используйте файлы `.env` для хранения секретов локально и добавляйте `.env` в `.gitignore`.
- Для демонстрации используйте открытые датасеты или синтетические данные.
- Перед коммитом проверяйте, что в файлах нет конфиденциальной информации.

## Что делать, если секрет случайно попал в репозиторий

1. Немедленно отзовите/перевыпустите скомпрометированный ключ/токен.
2. Удалите секрет из истории Git (например, с помощью `git filter-branch` или BFG Repo-Cleaner).
3. Сообщите преподавателю, если это может повлиять на безопасность курса.

## Контакт

При обнаружении проблем безопасности обращайтесь к преподавателю курса.
