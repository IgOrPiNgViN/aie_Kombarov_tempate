# Чеклист самопроверки проекта

Перед сдачей проекта проверьте выполнение следующих пунктов:

## Минимальные требования (обязательно)

- [ ] Проект запускается без ошибок
- [ ] Есть рабочий ML-сервис или демонстрация модели
- [ ] Заполнен `README.md` с инструкциями по запуску
- [ ] Код не содержит секретов (токены, пароли, ключи)

## Чеклист для оценки (каждый пункт = +1 балл)

1. [ ] **Структура проекта**: код организован по папкам (`src/`, `notebooks/`, `data/`, и т.д.)
2. [ ] **Документация**: заполнен `report.md` с описанием задачи, данных и экспериментов
3. [ ] **Воспроизводимость**: указаны все зависимости в `requirements.txt`, есть инструкции по запуску
4. [ ] **Версионирование данных**: используется DVC или аналог для управления данными
5. [ ] **Эксперименты**: используется MLflow/W&B или аналог для трекинга экспериментов
6. [ ] **Тестирование**: есть unit-тесты для ключевых функций
7. [ ] **CI/CD**: настроен GitHub Actions для автоматических проверок
8. [ ] **Контейнеризация**: проект можно запустить в Docker
9. [ ] **API**: реализован REST API для модели (FastAPI/Flask)
10. [ ] **Качество кода**: используется линтер (ruff/flake8), код соответствует PEP8
11. [ ] **Конфигурация**: параметры вынесены в конфиги (YAML/JSON), не захардкожены
12. [ ] **Логирование**: в коде используется logging вместо print

## Подсчёт баллов

- **Меньше 5 пунктов** — оценка 3
- **5-8 пунктов** — оценка 4  
- **9+ пунктов** — оценка 5
